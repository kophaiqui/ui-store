# Introduction

Not another UI library.
A pre-start design system for developers who want to ship without rebuilding the same foundation every time.

**That One UI** is not a package you install.
It is a copy-paste collection of components, color tokens, typography, and style presets that live inside your codebase — owned by you from day one.

Most UI libraries start with components.
That One UI starts before that.

## What it is

- **21 components** built on [Base UI](https://base-ui.com) primitives — accessible, unstyled at the core
- **11 styles per component** via a `styleConfig` prop — swap the visual layer without touching logic
- **Tailwind CSS** for all styling — works with any Tailwind project out of the box
- **TypeScript-first** — every component is fully typed, including the style config

## The styleConfig pattern

Every component accepts an optional `styleConfig` prop. The default style ships with the component in `styles/default.ts`. To change the look, pass a different config:

```tsx
import { UIButton } from "./Button";
import { glassStyle } from "./styles/glass";

<UIButton styleConfig={glassStyle} variant="solid">
  Click me
</UIButton>
```

The config is a plain object of Tailwind class strings. Swapping it changes every visual detail — colors, borders, shadows, radius — while the component's props, behavior, and accessibility stay exactly the same.

## All props are adjustable

Nothing is hardcoded. Every default value — labels, placeholder text, items arrays — is a prop you can override. Defaults exist only so the component renders meaningfully with zero configuration.

## Built on Base UI

[Base UI](https://base-ui.com) is the headless primitive layer built by the MUI team. It handles ARIA, keyboard navigation, focus management, and accessibility so you don't have to. That One UI adds the visual layer on top.

## A different idea

Most component libraries drop you at the component level and leave you to figure out the foundation — color tokens, typography scale, dark mode, a visual language that doesn't feel patched together.

That One UI ships the foundation.
Colors, fonts, motion patterns, and style presets are ready before you write your first component.

Browse what fits, copy what you need, leave with the pieces.
No setup. No config. No package to maintain.
Just a clear picture of your UI before it exists.

**Why it's different:**
- Foundation first — tokens, typography, and styles are ready before your first component
- Own your code — no black-box runtime, no locked-in abstraction
- Swap styles without touching logic — one prop changes the entire visual layer
- Consistent by default — every component shares the same tokens and language
- Built to evolve — change your brand, extend a component, or drop one entirely without rewriting anything
